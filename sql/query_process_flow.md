# 쿼리 평가 모듈

* 파서
* 옵티마이저
	* 플랜 생성
		* 인덱스 유무, 데이터 분산 또는 편향 정도, DBMS 내부 매개변수 등의 조건을 고려
		* 선택 가능한 많은 실행 계획을 작성
	* 비용 평가
* 카탈로그 매니저
	* 카탈로그(통계 정보) : DBMS의 내부 정보(테이블, 인덱스의 통계 정보 등)를 모아놓은 테이블들
	* 옵티마이저에 중요한 정보를 제공
		* 각 테이블의 레코드 수
		* 각 테이블의 필드 수와 필드의 크기
		* 필드의 카디널리티
		* 필드값의 히스토그램(어떤 값이 얼마나 분포되어 있는가)
		* 필드 내부에 있는 NULL 수
		* 인덱스 정보
	* 실제 테이블 정보와 카탈로그가 일치하지 않는 경우 문제가 생긴다.
		* 통계 정보를 생성하기 위한 시간이 필요할 수도 있음
	* 통계 정보 갱신 명령어
		* Oracle : ```exec DBMS_STATS.GATHER_TABLE_STATS(OWNNAME => [스키마 이름], TABNAME => [테이블 이름]);```
		* Microsoft SQL Server : ```UPDATE STATISTICS [테이블 이름]```
		* DB2 : ```RUNSTATS ON TABLE [스키마 이름].[테이블 이름];```
		* PostgreSQL : ```ANALYZE [스키마 이름].[테이블 이름];```
		* MySQL : ```ANALYZE TABLE [스키마 이름].[테이블 이름];```
* 플랜 평가
	* 옵터마이저의 여러 개의 실행 계획 중 최적의 실행 결과를 선택
